# ZLink Multi-Platform SDK
# ZLink 멀티플랫폼 SDK

The `sdk/` folder contains the core communication engines that drive the ZLink framework.
`sdk/` 폴더는 ZLink 프레임워크를 구동하는 핵심 통신 엔진들을 포함하고 있습니다.

Every SDK aims for "Self-Contained" logic. You can build your socket system by simply copying the `zlink` folder for each platform.
모든 SDK는 "자기 완결적" 로직을 지향하며, 각 플랫폼별 `zlink` 폴더를 복사하는 것만으로 소켓 시스템을 구축할 수 있습니다.

---

## 📂 SDK Structure
## 📂 SDK 구조

- **`server/zlink/`**: A high-performance Go-based server engine.
- **`server/zlink/`**: Go 기반의 고성능 서버 엔진입니다.
- **`client/python/zlink/`**: An asynchronous communication library based on Python's `asyncio`.
- **`client/python/zlink/`**: Python `asyncio` 기반의 비동기 통신 라이브러리입니다.
- **`client/unity/zlink/`**: A C# and plugin-based module optimized for the Unity engine.
- **`client/unity/zlink/`**: Unity 엔진에 최적화된 C# 및 플러그인 기반 모듈입니다.

---

## 🛠 Internal Protocol
## 🛠 내부 프로토콜

Each SDK has a `protocol/` subfolder.
각 SDK는 `protocol/` 하위 폴더를 가지고 있습니다.

The `Protocol` code generated by the `generator` is deployed here, ensuring that the engine and business logic (packets) always move as one.
제네레이터에 의해 생성된 `Protocol` 코드가 이곳에 배포되며, 엔진과 비즈니스 로직(패킷)이 언제나 일체화되어 움직이는 것을 보장합니다.
